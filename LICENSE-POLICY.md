# License Policy

This repository is licensed under `MPL-2.0`. The [LICENSE](LICENSE) file is
the controlling legal text. This document is a plain-language usage guide, not
an additional license.

## What MPL-2.0 Allows

- commercial and non-commercial use
- private internal use and modification
- distribution of unmodified or modified builds
- combining MPL-covered files with other files in a larger work

## What MPL-2.0 Requires When You Distribute Changes

- if you modify an MPL-covered source file and distribute that modified file or
  a build containing it, the MPL-covered source must remain available under
  `MPL-2.0`
- you must keep license notices intact in MPL-covered source files
- executable distributions must tell recipients how to obtain the corresponding
  MPL-covered source code

## Practical Rule

Think of the copyleft boundary as file-level:

- changes to files that already contain MPL-covered code stay under `MPL-2.0`
- separate new files in a larger work can use different terms, as long as you
  still comply with the MPL for the covered files

## Network Use

Running a server internally or as a hosted service does not by itself trigger a
source publication requirement under `MPL-2.0`. MPL obligations are primarily
about distributing covered source or executable copies.

## Historical Revisions

If an older revision was published under a different license, that earlier
revision keeps its original terms. This repository's current root license file
describes the license for the current revision.
